#输入流状态
- cin 是一个状态机，包括`good`,'fail(读取失败)','eof(文件结束)'三种状态，一旦进入`fail`, cin >> x将永远失败。
- 当 cin 读取失败进入`fail`状态时，输入数据仍然留在缓存区中，后续读取也全部失效。应该如下处理：
 ```
#include <iostream>
#include <limits>

cin.clear() //用以去除错误状态
cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n'); //清空一整行输入缓存
 
 ```
 - 实际工程中，使用 cin 处理错误太麻烦，建议使用 `getline+stoi/stod`，这样不会有缓冲区问题，不需要 ignore，更加稳定，建议统一输入模型：
 ```
 //input.h
 
 int readInt() {
    while (true) {
        string line;
        getline(cin, line);//从标准输入读取数据直到换行符，换行符不进入 line 中
        try {
            return stoi(line);//尝试将 line 转为整数，若不合要求，进入 catch 块
        } catch (...) { // ...表示通配所有异常类型
            cout << "Invalid input!\n";
        }
    }
}
 ```

#从文件到数据结构
- 从文件读取原始数据，需要用输入流（`ifstream` 或 `istringstream`）将文本/二进制数据解析、转换为程序中的数据结构（如对象、向量）。  
- 相反，要将程序中的数据存回文件，需要用输出流（`ofstream`），把数据结构转换成文本或二进制格式保存。
- 可以说，流是数据的通道，在“外部存储”与“内部数据结构”之间搬运转换数据
	文件 -> 输入流 -> 数据结构 -> 输出流 -> 文件
```
//文件读取
#include <fstream>
#include <sstream>

std::ifstream fin("data.txt"); //打开文件，作为输入流与程序连接
std::string line; //用于存储流的每行数据
while (std::getline(fin, line)) { //将文件流逐行读取到 line 中
    std::istringstream iss(line); //将 line 包装为 string 流，可以像 cin 一样解析
    int id;
    std::string name;
    iss >> id >> name;  // 从流中提取数据转换为相应类型
}

//贴一个代码

while (!iss.eof()) {
	string courseStr，gradeStr, creditStr, weightStr;
	if (!std::getline(iss, courseStr, ',')) break; //按照指定字符读取
	if (!std::getline(iss, gradeStr, ',')) break;
	if (!std::getline(iss, creditStr, ',')) break;
	if (!std::getline(iss, weightStr, ',')) break;
	double grade = std::stod(gradeStr);
	double credit = std::stod(creditStr);
	double weight = std::stod(weightStr);
	Score score(courseStr, grade, credit, weight);
	student.addScore(score);
}
students.push_back(student);
```

#死循环常见情况处理
- **cin/fail状态** 一个主文件里面有八百个死循环，大部分是由于 cin 流导致的，摒弃（cin输入失败未清理缓存，与getline混用导致残留换行符，注意 cin 读取的数据不会清楚缓存区中的换行符）
- **正确终止** while (flag) 注意何时更新 flag 何时 break，如果是 while (true) 一定要有 break
- **循环逻辑，避免跳转*switch(choice) / case / defalut, 后都要接 break，否则会自动跳转下一个情况（如果存在）
- **字段解析** 多字段输入出错时要消耗错误字段，否则下一轮输入会死循环
